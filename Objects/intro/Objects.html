OBJECTS IN TS

Objects in TS are just like JS, except in TS we use tpye annotation for the key value pairs inside the object

This is especially useful when we use functions tor create new objects

Objects can be typed by declaring what the object should look like in the annotation

Accessing a property that isn't defined or performing operations without keeping types in mind will throw errors!

Let's try this out, let's declare this function below and in the parameter list let's say it's expecting an object to be passed in:

function printName(person: {first: string}){}

At this point above, if we pass in something like:

printName({})

TS in the IDE will say 'hey! you said printName needed the property "first" what's going on here programmer man??'

Then we mess with typescript again and pass in:

printName({first: 2})

Then typescript is like 'okay nice you passed in the first property good job, WAIT! you passed in a number?? come on man you said in the declaration annotation first needed to be a string!!'

printName({first: "Luke"})

Now TS throws no errors, we're good

But let's say we want both the first and last name printed

printName({first: string, last: string})

printName({first: "Luke"})

Now TS says 'WAIT! You're missing the last property

printName({first: "Luke", last: "Sorrenti"})

function printName(person: { first: string;

last: string}): void {

console.log(`Hi! I'm {person.first} {person.last}` );

}

printName({first: "Luke", last: "Sorrenti"});

"Hi! I'm Luke Sorrenti" is logged to the console

More object types!

We just saw we an annotate a function parameter type, to be an object, we can do the same for a return type, like this function should only ever return a boolean or string, or whatever data type

EXAMPLE:

Let's use a coordinate variable where x and y should always be numbers, if we're trying to navigate using coordinates we wouldn't want to pass in a boolean or any other data type other than an integer

What is TypeScript?

To answer this question, we need to talk JavaScript. JavaScript has a few quirks, shortcomings, oddities. Like 'null * 213' returns 0, or multiply 'undefined * 8976', or if we try to access a property of an object that doesn't exist, JS says 'undefined', it doesn't throw any errors, just throws unexpected undefined

The quirks are many because JS was created very quickly, just to add some scripting. But JS now is a behemoth, it's THE browser language.

TypeScript improves upon JS by adding 'TYPES', TypeScript is known as a 'superset' of JS, meaning all JS code is considered valid, the difference is TypeScript adds types

So what does all this actually mean? Well TypeScript performs something called 'Static Checking', which is a feature of many other programming languages, but essentially it means that it will detect errors in our code without running it

It doesn't actually execute our code, that's JavaScripts job, TypeScript is a static type checker, which means it does this 'pre-runtime' checking on the basis of types, or the 'types of data' in our program, it knows if something is a number, or boolean, or string, or an array, or object.

Why does this matter? The main reason is that it greatly helps identifying errors in our code, and more importantly it helps identify them BEFORE our code even runs, it analyzes what we do as we type, and only exists in development, it will become regular JavaScript that can run in the browser, or node.js

Our first type annotation

So we've established the obvious point that there are types in TypeScript, there are many of these types and we'll dive into all of them

But first let's talk about where we apply types in our TypeScript code there's many different situations from things like functions, i.e: this function must return a boolean, or an array of booleans, or a number, string, you get the idea

We can also say 'This function accepts two numbers and returns one number'

We can get very detailed about the patterns, shapes of objects, like this object must have a specific property called 'colors', and 'colors' must be set to an array of strings

Or we can do something very simple by saying 'hey TS, this variable is a string'

We can do this very easily, all we do is 'const myVariable: string = "so awesome!"'

And after running the tsc command our JS file will display ' const myVariable = "so awesome!"'

We just added a type! This is called TYPE ANNOTATION

The syntax is: let myVar: type = value

And the type is always lowercase, no capital S for string

Also we can reassign variables is TS if it's the same type, for ex we could do 'myVariable = "TypeScript is awesome"' and our variable from earlier updates, similar to how 'let' works in JS

For ex: let movieTitle: string = "Amadeus"; movieTitle = "Arrival"

Now movieTitle = Arrival, but if we change it to be a number

movieTitle = 9

Type 'number' is not assignable to type 'string'.

So TS is fine with us changing movieTitle from Amadeus to Arrival, but once we try to change movieTitle to a number, TS throws the error

Something similar happens if we try to use a method on the wrong data type, like trying to use .toUpperCase on a number

Working with numbers and booleans

So just like in JS we have a number data type, some languages have things like 'floats' ,'int' and others but in TypeScript as well as JavaScript numbers are just numbers

In TS we use the type annotation 'number' all lowercase

When we tell TS 'this specific variable is a number, it should always be a number, the number can be reassigned, but it must be a number'

We also have 'boolean' as a type, that's the same concept as number, 'boolean' must be lowercase, and if the value of the variable changes from true to false, that's fine, but it must be a boolean

Type Inference

Let's go over type Inference, type inference refers to the TypeScript compilers ability to infer types from certain values in your code.

TypeScript can remember a value's type even if you didn't provide a type annotation, and will enforce that original type set to that variable from then on. Ex:

//Creating a variable with a value but without type annotation

let x = 27

x = "twenty-seven"

//ERROR - Type 'string' is not assignable to type 'number'

Most of the time when we write a variable, we can rely on type inference, but there ARE situations where we must annotate variables, and we'll learn more about that later

//type inference

let tvShow = "Initial D";

tvShow = "Breaking Bad"

//no problem, tvShow is now breaking bad

tvShow = false

//problem, Type 'boolean' is not assignable to type 'string'.ts(2

Above we did not use type inference, we could've but we let TS infer it for us.

The type: 'Any':

The 'any' type annotation is an escape hatch, it allows us to turn off type checking for a specified variable, Note: this defeats the purpose of using TypeScript so learn when / when not to use it!
